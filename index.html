Codage de Huffman

Principe • Technique • Avantages • Inconvénients • Exemple

Principe

Le codage de Huffman est un algorithme de compression sans perte. Il attribue des codes binaires courts aux symboles fréquents et des codes plus longs aux symboles rares. Ainsi, la longueur totale du message compressé est réduite.

Technique

1️⃣ Compter la fréquence de chaque caractère dans le message.
2️⃣ Construire un arbre binaire de Huffman en fusionnant les nœuds les moins fréquents.
3️⃣ Attribuer un code binaire : 0 pour la branche gauche, 1 pour la droite.
4️⃣ Lire les codes binaires pour chaque caractère.
5️⃣ Remplacer le texte original par sa version compressée.

Avantages

✔️ Compression efficace pour les fichiers texte ou avec symboles redondants.
✔️ Algorithme simple à implémenter.
✔️ Sans perte : on peut retrouver exactement le message original.

Inconvénients

⚠️ Moins efficace si tous les symboles apparaissent avec la même fréquence.
⚠️ Demande un dictionnaire de codes (arbre) pour décoder.
⚠️ Pas optimal pour des données déjà compressées.

Exemple simple

Message à compresser : "ABRACADABRA"

Étape 1 : Fréquences

  • A → 5
  • B → 2
  • R → 2
  • C → 1
  • D → 1

Étape 2 : Codes attribués

Codage selon l'arbre :
A = 1
B = 01
R = 001
C = 0001
D = 0000

Étape 3 : Encodage

"ABRACADABRA" devient :
1 01 001 1 0001 1 0000 1 01 001 1

➝ Le message compressé occupe moins de bits que l’ASCII classique.
